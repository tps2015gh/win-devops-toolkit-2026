## 📋 ข้อกำหนดเบื้องต้น

ก่อนเริ่มใช้งาน ต้องเตรียมสิ่งต่อไปนี้:

| รายการ | รายละเอียด | สถานะ |
|--------|-----------|-------|
| **MCP Server** | ไฟล์ `db_mcp_server.exe` ที่คอมไพล์แล้ว | ✅ จำเป็น |
| **ฐานข้อมูล** | MySQL หรือ MariaDB ที่กำลังทำงานอยู่ | ✅ จำเป็น |
| **Qwen Code CLI** | ติดตั้งและตั้งค่าเรียบร้อยแล้ว | ✅ จำเป็น |
| **ข้อมูลเชื่อมต่อ** | ชื่อผู้ใช้, รหัสผ่าน, โฮสต์, พอร์ต | ✅ จำเป็น |

### การตรวจสอบข้อกำหนด

```powershell
# 1. ตรวจสอบว่า db_mcp_server.exe มีอยู่
dir C:\dev\abcd_setup_server\db_mcp_server.exe

# 2. ตรวจสอบว่า MySQL กำลังทำงาน
Get-Service -Name MySQL*

# 3. ตรวจสอบเวอร์ชัน Qwen
qwen --version
```

## 🔧 การตั้งค่า

### ขั้นตอนที่ 1: สร้างไฟล์ตั้งค่า MCP

#### วิธี A: ตั้งค่าระดับโปรเจกต์ (แนะนำ)

สร้างไฟล์ `.qwen/mcp.json` ในรูทของโปรเจกต์:

```json
{
  "mcpServers": {
    "database": {
      "command": "C:\\dev\\abcd_setup_server\\db_mcp_server.exe"
    }
  }
}
```

#### วิธี B: ตั้งค่าระดับ Global

**Windows:** สร้างหรือแก้ไข `%APPDATA%\Qwen\mcp.json`

**macOS/Linux:** สร้างหรือแก้ไข `~/.qwen/mcp.json`

```json
{
  "mcpServers": {
    "database": {
      "command": "C:\\dev\\abcd_setup_server\\db_mcp_server.exe"
    }
  }
}
```

### ขั้นตอนที่ 2: แก้ไขพาธให้ตรงกับระบบของคุณ

| ตำแหน่งของคุณ | พาธใน Config |
|---------------|-------------|
| โปรเจกต์ปัจจุบัน | `C:\\dev\\abcd_setup_server\\db_mcp_server.exe` |
| โฟลเดอร์ที่กำหนดเอง | `D:\\tools\\db_mcp_server.exe` |
| ใน PATH | `db_mcp_server.exe` |

> ⚠️ **หมายเหตุ Windows:** ใช้ backslash คู่ (`\\`) ใน JSON

### ขั้นตอนที่ 3: รีสตาร์ท Qwen Code

หลังจากเพิ่มการตั้งค่าแล้ว **ให้รีสตาร์ท Qwen Code CLI** เพื่อให้การเปลี่ยนแปลงมีผล

```bash
# ปิด Qwen แล้วเปิดใหม่
qwen
```

## ✅ ตรวจสอบการเชื่อมต่อ

### การทดสอบที่ 1: ตรวจสอบเครื่องมือที่มี

**ถาม Qwen:**
```
คุณมีเครื่องมือ MCP อะไรบ้าง?
```

**คำตอบที่คาดหวัง:**
```
ฉันมีเครื่องมือ Database MCP ต่อไปนี้:
- list_databases: แสดงรายการฐานข้อมูล
- list_tables: แสดงรายการตาราง
- get_table_schema: ดูโครงสร้างตาราง
- query_data: ค้นหาข้อมูล
- get_table_stats: ดูสถิติตาราง
- get_database_info: ดูข้อมูลเซิร์ฟเวอร์
- search_data: ค้นหาแบบซับซ้อน
- get_foreign_keys: ดู Foreign Key
- get_indexes: ดู Index
- aggregate_data: คำนวณ Aggregate
- switch_database: เปลี่ยนฐานข้อมูล
```

### การทดสอบที่ 2: แสดงรายการฐานข้อมูล

**ถาม Qwen:**
```
แสดงฐานข้อมูลทั้งหมด
```

**คำตอบที่คาดหวัง:**
```json
{
  "databases": ["mydb", "testdb", "production"],
  "count": 3
}
```

```
| id        | int(11)       | NO   | PRI  | NULL       | auto_increment |
| username  | varchar(50)   | NO   |      | NULL       |                |
| email     | varchar(100)  | NO   | UNI  | NULL       |                |
| password  | varchar(255)  | NO   |      | NULL       |                |
| created_at| timestamp     | YES  |      | CURRENT_TIMESTAMP |         |
```

## 🔒 หมายเหตุความปลอดภัย

### การเข้าถึงแบบอ่านอย่างเดียว

MCP server **อนุญาตเฉพาะ**:
- คำสั่ง `SELECT`
- คำสั่ง `SHOW`
- คำสั่ง `DESCRIBE`
- คำสั่ง `EXPLAIN`

**การดำเนินการที่ถูกปิดกั้น:**
- `INSERT`, `UPDATE`, `DELETE`
- `DROP`, `TRUNCATE`, `ALTER`
- การเขียนข้อมูลทุกประเภท

### การจัดการข้อมูลประจำตัว

- ข้อมูลประจำตัวถูกส่งตรงไปยัง MySQL driver
- รหัสผ่าน **ไม่ถูกบันทึก** หรือเก็บไว้ในล็อก
- ควรใช้ผู้ใช้ฐานข้อมูลที่มีสิทธิ์ต่ำสุดเท่าที่จำเป็น

## 🐛 การแก้ปัญหา

### ปัญหา: ไม่พบ MCP Server

**ข้อผิดพลาด:** `Failed to start MCP server: db_mcp_server.exe not found`

**วิธีแก้:**
1. ตรวจสอบว่าพาธใน `.qwen/mcp.json` ถูกต้อง
2. ใช้พาธแบบเต็มพร้อม backslash คู่ (Windows):
   ```json
   "command": "C:\\\\dev\\\\abcd_setup_server\\\\db_mcp_server.exe"
   ```
3. ตรวจสอบว่าไฟล์ executable มีอยู่และไม่ถูกปิดกั้นโดยโปรแกรมป้องกันไวรัส

---

### ปัญหา: ไม่พบ mysql.exe

**ข้อผิดพลาด:** `mysql.exe not found. Please ensure MySQL/MariaDB is installed`

**วิธีแก้:**
1. เพิ่ม MySQL เข้า PATH:
   ```powershell
   setx PATH "%PATH%;C:\xampp\mysql\bin"
   ```
2. หรือตั้งค่า `MYSQL_HOME`:
   ```powershell
   setx MYSQL_HOME "C:\xampp\mysql\bin"
   ```
3. รีสตาร์ท Qwen Code หลังการเปลี่ยนแปลง

---

### ปัญหา: การเชื่อมต่อถูกปฏิเสธ

**ข้อผิดพลาด:** `connection failed: dial tcp 127.0.0.1:3306: connectex: No connection could be made`

**วิธีแก้:**
1. ตรวจสอบว่าเซอร์วิส MySQL/MariaDB กำลังทำงาน:
   ```powershell
   Get-Service -Name MySQL*
   ```
2. ตรวจสอบโฮสต์และพอร์ตในการเชื่อมต่อ
3. ตรวจสอบว่า firewall อนุญาตการเชื่อมต่อบนพอร์ต 3306

---

### ปัญหา: Access Denied

**ข้อผิดพลาด:** `Access denied for user 'root'@'localhost'`

**วิธีแก้:**
1. ตรวจสอบชื่อผู้ใช้และรหัสผ่าน
2. ตรวจสอบสิทธิ์ของผู้ใช้:
   ```sql
   SHOW GRANTS FOR 'root'@'localhost';
   ```
3. มอบสิทธิ์ที่จำเป็นหากต้องการ

---

### ปัญหา: ไม่มีเครื่องมือ tersedia

**อาการ:** Qwen ไม่แสดงรายการเครื่องมือฐานข้อมูล

**วิธีแก้:**
1. ตรวจสอบว่าไวยากรณ์ JSON ใน `.qwen/mcp.json` ถูกต้อง
2. รีสตาร์ท Qwen Code CLI โดยสมบูรณ์
3. ตรวจสอบล็อกของ Qwen สำหรับข้อผิดพลาดการเชื่อมต่อ MCP
4. ตรวจสอบว่า MCP server ทำงานได้ด้วยตนเอง:
   ```powershell
   C:\dev\abcd_setup_server\db_mcp_server.exe
   ```

```
ผู้ใช้: แสดงฐานข้อมูลทั้งหมด
Qwen: [เรียก list_databases] พบ: mydb, testdb, production

ผู้ใช้: เปลี่ยนไปใช้ฐานข้อมูล production
Qwen: [เรียก switch_database] เปลี่ยนไปใช้ 'production' สำเร็จ

ผู้ใช้: แสดงตารางทั้งหมด
Qwen: [เรียก list_tables] พบ: users, orders, products, categories

ผู้ใช้: ตาราง orders มีคอลัมน์อะไรบ้าง?
Qwen: [เรียก get_table_schema] ส่งคืน: id, customer_id, total, status, created_at, updated_at

ผู้ใช้: แสดงคำสั่งซื้อที่รอการดำเนินการที่เกิน $500
Qwen: [เรียก search_data] ส่งคืนผลลัพธ์ที่กรองแล้ว

ผู้ใช้: คำนวณยอดรวมทั้งหมดแยกตามสถานะ
Qwen: [เรียก aggregate_data] ส่งคืนผลรวมแบบจัดกลุ่ม
```
